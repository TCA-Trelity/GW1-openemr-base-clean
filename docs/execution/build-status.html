Wave M — multi-turn agent emphasis + thought-partner guardrails

Merged to main 2026-07-11 · 9/9
Early-submission feedback: "the eval suite is one of the strongest in the batch … however, the rubric explicitly excludes a 'report generator' as the core interface, and the ARCHITECTURE emphasis on a pre-assembled one-page brief … reads closer to that pattern than to the multi-turn tool-invoking agent the rubric asks for; … promoting that conversational surface to a primary role in both the doc and the demo would make that clear."

The chat already is that agent — persisted history, 4-round tool loop, six patient-scoped tools, verbatim-verified citations — but every graded artifact points at the brief: 0 of 11 evals exercise a conversation, Bruno never sends a second message, the panel ships chat closed behind a floating button, and USERS.md traces multi-turn to one use case. Wave M moves the weight, and adds the guardrail the prompt layer lacks: thought partner, never prescriber — cited record facts · engine/guideline output with attribution · questions worth considering. The decision stays with the physician.

Deploy pipeline — converged: e2e green, tagged, baselines committed

Submission-ready

Every sidecar deploy since the chat tools landed failed with TS2307 ./tools/index.js. Root cause (S3.7): an unanchored tools/ export-ignore in .gitattributes matched sidecar/src/chat/tools/ at any depth, and Railway builds from the GitHub source archive, which honors export-ignore — git checkouts (CI, local) don't. The deploy context was missing eight committed files; no cache busting could restore them. Fixed by anchoring /tools/, adding a sidecar/** -export-ignore carve-out, and a CI export-parity job so a tarball-only breakage can never hide again.
